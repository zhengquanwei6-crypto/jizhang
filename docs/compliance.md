# CoupleSpaceAI 合规与发布清单

## 隐私合规（中国大陆）

### 1. 个人信息收集清单

| 数据类型 | 收集目的 | 保存期限 | 是否必需 |
|----------|----------|----------|----------|
| 手机号/邮箱 | 账号注册与登录 | 账号有效期内 | 是 |
| 情侣绑定关系 | 关联双方数据 | 账号有效期内 | 是 |
| 昵称/头像 | 展示情侣身份 | 账号有效期内 | 是 |
| 账单数据 | 共同记账功能 | 账号有效期内 | 否 |
| 聊天消息 | 情侣通信功能 | 账号有效期内 | 否 |
| 定位坐标 | 地图足迹功能 | 用户删除前 | 否 |
| 打卡记录 | 目标打卡功能 | 账号有效期内 | 否 |
| 推送 Token | 消息推送 | 卸载前 | 否 |

### 2. 权限申请时机

| 权限 | 申请时机 | 拒绝后果 |
|------|----------|----------|
| 网络 | App 启动 | 无法使用 |
| 通知 | 用户主动触发设置页 | 无法推送 |
| 定位 | 用户点击地图足迹入口 | 无法使用地图 |
| 存储/相册 | 用户选择发送图片 | 无法发送图片 |
| 相机 | 用户选择拍照 | 无法拍照发送 |

### 3. 隐私协议必须包含

- [ ] 数据收集范围说明
- [ ] 第三方 SDK 清单（极光、高德、Supabase、Coil、Lottie）
- [ ] 定位数据处理说明（不上传第三方、不后台持续定位）
- [ ] 数据存储与删除机制
- [ ] 账号注销流程
- [ ] 未成年人保护声明

### 4. 首次启动弹窗

必须在用户首次启动时弹出隐私协议弹窗：
- 阅读隐私政策链接
- 同意/拒绝两个按钮
- 拒绝则退出 App

## 构建配置

### 多 flavor

| flavor | applicationIdSuffix | 用途 |
|--------|---------------------|------|
| dev | .dev | 开发测试 |
| staging | .staging | 预发布测试 |
| prod | (无) | 生产上线 |

### R8 / ProGuard

需配置以下库的 keep 规则：
- `io.supabase` (Supabase SDK)
- `io.retrofit2` (Retrofit)
- `io.room` (Room)
- `io.coil` (Coil)
- `com.airbnb.lottie` (Lottie)
- `cn.jiguang` (极光)
- `com.amap.api` (高德)

### 应用签名

```properties
# keystore.properties (不提交到 git)
storePassword=<密码>
keyAlias=<别名>
keyPassword=<密码>
storeFile=<keystore 路径>
```
